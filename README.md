## Featured Project

### Revolv 360 â€“ Android Weight Tracking Application

Revolv 360 is a mobile weight tracking application that allows users to log daily weight entries, view historical trends, track progress toward goals, and receive motivational feedback. The project was originally developed in CS 360 and later enhanced as part of my CS 499 capstone to demonstrate growth across multiple computer science competencies.

---

## Capstone Enhancements

### Enhancement One: Software Engineering and Design

For my first enhancement, I improved the software engineering and design aspects of the Revolv 360 application by implementing a full dark mode toggle. The original version of the app relied on a single static theme, which limited accessibility and did not reflect modern user interface standards. I redesigned the theme architecture so users could switch between light and dark mode, with all screens updating instantly based on the selected preference.

This enhancement required refactoring UI components, updating color resources, managing shared theme state, and ensuring that user preferences persist across sessions. Completing this enhancement allowed me to apply software design principles while improving maintainability and usability. It also reflects my growth as a front-end developer who balances technical implementation with user-centered design.

---

### Enhancement Two: Algorithms and Data Structures

The second enhancement focused on expanding application functionality through algorithmic logic and data handling. I implemented a weekly weight change badge that calculates a userâ€™s progress over a seven-day period. While the original application stored weight entries, it did not provide analytical feedback based on that data.

To implement this feature, I designed logic to identify the correct date range, retrieve relevant entries, perform accurate calculations, and update the user interface dynamically as data changed. This process strengthened my understanding of algorithms, list processing, conditional logic, and data validation. The enhancement shows how well-designed algorithms and appropriate data structures can directly improve functionality and user engagement.

---

### Enhancement Three: Databases

For the third enhancement, I expanded the database functionality of the application by adding support for notes associated with each weight entry. In the original version, the database stored only a single numeric value per day, which limited the usefulness of the stored data.

I modified the data model to include a notes field and updated the logic for creating, editing, and retrieving entries. This enhancement shows my ability to design and evolve database structures while maintaining data integrity and reliability. It also demonstrates my understanding of how data flows from user input through persistent storage and back into the user interface.
